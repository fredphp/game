## 二、A 级活动设计（每月循环）

### 2.1 「乱世争锋」— 世界BOSS挑战

| 参数 | 数值 |
|------|------|
| 持续时间 | 7天 |
| 参与方式 | 全服玩家共同攻击BOSS |
| 奖励机制 | 个人伤害排名 + 全服击杀里程碑 |
| 消耗 | 每次挑战消耗体力×20 |

**奖励池：**

| 排名 | 钻石 | SSR碎片 | 限定道具 |
|------|------|---------|---------|
| Top 1 | 5,000 | 50 | 限定称号"天下第一" |
| Top 2~10 | 3,000 | 30 | 限定头像框 |
| Top 11~50 | 1,500 | 15 | — |
| Top 51~100 | 800 | 8 | — |
| Top 101~500 | 300 | 3 | — |
| 参与奖 | 100 | 1 | 金币×2万 |

**全服里程碑：**
- 全服累计伤害 100亿: 全服钻石×100
- 全服累计伤害 500亿: 全服SSR碎片×5
- 击败BOSS: 全服十连券×1 + 钻石×500

### 2.2 「英雄试炼」— 爬塔挑战

| 参数 | 数值 |
|------|------|
| 持续时间 | 14天 |
| 层数 | 200层 |
| 玩法 | 3队编队分别挑战3条路线，最高层数求和 |
| 重置 | 每次活动开始重置进度 |

**关卡设计：**
- 1~50层: 普通难度 (R/SR敌人)
- 51~100层: 困难难度 (SR/SSR敌人, 阵营克制关键)
- 101~150层: 噩梦难度 (全SSR敌人, 属性碾压)
- 151~200层: 地狱难度 (机制怪, 需要特定阵容)

**奖励（按最高层数发放）：**

| 里程碑 | 奖励 |
|--------|------|
| 30层 | 钻石×200 + 突破石×20 |
| 60层 | 十连券×1 + SR自选券×1 |
| 100层 | 钻石×800 + SSR碎片×20 |
| 150层 | 十连券×2 + 限定称号"试炼者" |
| 200层(通关) | 限定SSR角色 + 十连券×5 + 限定皮肤 |

### 2.3 「诸侯会盟」— 联盟对抗赛

| 参数 | 数值 |
|------|------|
| 持续时间 | 10天 |
| 形式 | 32强淘汰赛 |
| 参与门槛 | 联盟等级≥5, 成员≥20人 |
| 战斗 | 城池争夺(自动战斗) |

**赛程：**
```
Day 1~2: 小组赛 (8组×4联盟, 组内循环)
Day 3~4: 16强淘汰赛
Day 5~6: 8强赛
Day 7: 4强赛
Day 8: 半决赛
Day 9: 决赛 + 全服观战
Day 10: 颁奖 + 战利品兑换
```

**联盟排名奖励：**

| 名次 | 联盟奖励(人均) | 个人MVP奖励 |
|------|-------------|------------|
| 冠军 | 钻石×3000 + 限定SSR + 限定联盟旗帜 | UR武器碎片×30 |
| 亚军 | 钻石×2000 + SSR碎片×50 | SSR碎片×30 |
| 季军 | 钻石×1500 + SSR碎片×30 | SSR碎片×20 |
| 4~8名 | 钻石×800 + SSR碎片×15 | SR碎片×20 |
| 16强 | 钻石×400 + SR碎片×10 | — |
| 参与 | 钻石×200 + SR碎片×5 | — |

## 三、B 级活动设计（每周轮换）

### 3.1 周末循环活动矩阵

| 星期 | 活动名称 | 内容 | 核心奖励 |
|------|---------|------|---------|
| 周一~周三 | 「精炼大师」 | 装备强化成功率+20%，材料掉率+50% | 突破石、精炼石 |
| 周三~周五 | 「英雄盛宴」 | 抽卡UP概率+10%，十连额外送1抽 | 招募令、碎片 |
| 周五~周日 | 「金币大作战」 | 金币副本产出×2，金币商城限时开放 | 金币、经验书 |
| 周六~周日 | 「竞技盛典」 | 竞技场胜场奖励×2，段位挑战奖励加倍 | 竞技币、钻石 |
| 每日轮换 | 「悬赏任务」 | 6个随机悬赏任务，完成后开宝箱 | 随机材料、钻石 |

### 3.2 「秘境探索」— 限时副本

| 参数 | 数值 |
|------|------|
| 开放时间 | 每天 12:00~14:00, 20:00~22:00 |
| 次数 | 每日3次 |
| 难度 | 每周轮换 (普通→困难→噩梦) |
| 掉落 | 当前周对应材料掉率+100% |

### 3.3 「天命预言」— 押注活动

| 参数 | 数值 |
|------|------|
| 持续时间 | 3天 |
| 玩法 | 对下一期限定池UP角色进行押注 |
| 押注币 | 使用竞技币（不可充值获得） |
| 奖励 | 猜中: 返还3倍押注币 + 额外钻石 |

> **设计目的**: 增加社区讨论热度，为下一期限定池预热

## 四、S 级活动设计（季度大型）

### 4.1 周年庆活动（示例）

```
持续时间: 21天
主题: "九州一统·周年盛典"

Week 1: 回归福利周
├── 回归玩家送SSR自选 + 十连×5
├── 全服登录送钻石×500/天
└── 老带新: 邀请回归各得SSR碎片×20

Week 2: 消费返还周
├── 累充返还: ¥6→60钻, ¥30→600钻, ¥68→1200钻, ¥648→15000钻
├── 消费排行: Top100送限定UR
└── 限时商店: 稀有道具打折50%

Week 3: 全服挑战周
├── 全服共建: 累计击败BOSS解锁全服奖励
├── 联盟排行榜: 冠军联盟全送限定SSR
├── 个人排行榜: 综合战力/城池数/击杀数
└── 抽奖转盘: 消费积分兑换抽奖次数

Week 4 (前3天): 闭幕庆典
├── 最终BOSS: 全服合力击败
├── 倒计时奖励: 每天送十连×1
└── 下赛季预告 + 预约奖励
```

## 六、活动数据结构

```sql
-- 活动配置表
CREATE TABLE `activity_configs` (
    `id`              BIGINT       NOT NULL AUTO_INCREMENT,
    `activity_key`    VARCHAR(32)  NOT NULL COMMENT '活动标识 world_boss/tower/alliance_war...',
    `name`            VARCHAR(64)  NOT NULL,
    `type`            TINYINT      NOT NULL COMMENT '1=S级 2=A级 3=B级 4=C级',
    `start_time`      DATETIME     NOT NULL,
    `end_time`        DATETIME     NOT NULL,
    `config`          JSON         NOT NULL COMMENT '活动参数配置',
    `reward_config`   JSON         NOT NULL COMMENT '奖励配置',
    `status`          TINYINT      NOT NULL DEFAULT 1,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_key` (`activity_key`),
    KEY `idx_time` (`start_time`, `end_time`)
) COMMENT='活动配置表';

-- 用户活动进度表
CREATE TABLE `user_activity_progress` (
    `id`              BIGINT       NOT NULL AUTO_INCREMENT,
    `user_id`         BIGINT       NOT NULL,
    `activity_key`    VARCHAR(32)  NOT NULL,
    `progress`        JSON         NOT NULL DEFAULT '{}' COMMENT '进度数据(JSON)',
    `score`           BIGINT       NOT NULL DEFAULT 0 COMMENT '活动积分',
    `reward_claimed`  VARCHAR(1024) NOT NULL DEFAULT '' COMMENT '已领奖励位图',
    `extra_data`      JSON         DEFAULT NULL COMMENT '扩展数据',
    `created_at`      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    `updated_at`      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_user_activity` (`user_id`, `activity_key`),
    KEY `idx_score` (`activity_key`, `score`)
) COMMENT='用户活动进度表';

-- 活动排行榜表
CREATE TABLE `activity_rankings` (
    `id`              BIGINT       NOT NULL AUTO_INCREMENT,
    `activity_key`    VARCHAR(32)  NOT NULL,
    `user_id`         BIGINT       NOT NULL,
    `nickname`        VARCHAR(64)  DEFAULT '',
    `guild_name`      VARCHAR(64)  DEFAULT '',
    `score`           BIGINT       NOT NULL DEFAULT 0,
    `rank_value`      BIGINT       NOT NULL DEFAULT 0 COMMENT '排行数值(伤害/层数/等)',
    `extra_info`      JSON         DEFAULT NULL,
    `updated_at`      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_activity_user` (`activity_key`, `user_id`),
    KEY `idx_rank` (`activity_key`, `rank_value` DESC)
) COMMENT='活动排行榜表';
```
